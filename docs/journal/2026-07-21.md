## implementation, stage 1 — config and the project set

Implementation agent (Claude Code) session against the converged work order. Stage 1 landed:

No cached anything; `TestSourceConsultedFresh` pins the promise.

## implementation, stage 2 — the contract turns project-scoped

The breaking stage, landed as one green movement:

- `specs/ranger.yml`: every path under `/projects/{project}` with a shared parameter, a `404` (`notFound` → `errorResponse`) on every scoped operation, `GET /projects` → `projectIndex {projects: [{name, available, error?}], default}`. `make generate` regenerated both sides; ogen disambiguates same-schema statuses per operation, so the handlers' 400s became `CreateItemBadRequest`/`RetitleItemBadRequest`/`RenameToSlugBadRequest` and the three ItemLanded ops got per-op `…NotFound` types (plain `*api.ErrorResponse` remains the 404 for the board-returning ops).
- `internal/server`: `Server{projects *Projects}`; a `resolve()` helper maps `UnknownProjectError` → 404 body and lets a config-source failure ride as the request's plain error (heals on next good save). `freshBoard(w, project)` became a function; `board.project` now carries the configured name. `CreateItem` gained the fresh-`Load()` preflight between resolution and `CreateDraft`. `NewError`/`asConflict` untouched per the work order.
- `assets.go`: `ProjectAssets(projects)` — leading segment resolved through the source, then delegation to the untouched `Assets`; unknown project 404, source failure surfaces 500 plainly.
- `cmd/ranger/serve.go`: `config.Synthesize(root, port)` (the shared `normalize` extracted in `internal/config` — one grammar, both callers, errors tagged with an `origin`), `serveBootstrap` (synthesize → resolve → startup `Load()` fail-fast → mux), and `buildMux` — the assembly the daemon will share in stage 4.
- UI: `api.ts` → `makeApi(project)` factory carrying `project` on the object (so modals take one `api` prop), plus unscoped `fetchProjects()`. `App` split: route on `projectFromPath` (`ui/src/project.ts`, no decoding — slugs need none), `DefaultRedirect` consults the index and `location.replace`s to `/p/{default}`, `ProjectBoard` holds the old body with the bound api memoized off the URL project. `markdown.ts` → `makeRoadmapUrl(project)`: resolved-path containment via `new URL` against the `/roadmap/{project}/` base — pathname-prefix check after real URL normalization, inert (`""`) on escape.
- Acceptance tests per the work order: contract census over the parsed spec (`internal/api/census_test.go`); generated-client-through-generated-router round trip (read + mutation + 404) in `TestHTTPRoundTrip`; `TestProjectAssets` cross-project assertions (per-project bytes, unknown 404, four traversal spellings never serving b's bytes under a's name) with the client-side spellings (`../`, `..\`, `%2e%2e`, `.%2e`, host-change) pinned in vitest; `TestDegradationAndHeal` (index flags, board error, capture refusal with byte-for-byte tree comparison, on-disk heal recovering index and board on the same instance); serve's fail-fast tier in `cmd/ranger/serve_test.go`.

Green: full Go suite + vet, 26 UI tests, `make build`, plus a live smoke test (`serve --port 4199`: index, scoped board, 404 message, `/p/ranger`, asset 404).

Terminus on the stage: clean, one advisory — `config.Synthesize` renamed to `config.New` per the constructor convention (applied; suite re-run green). Held for Michael's review; approved, along with the pure-model testing strategy for stage 3 (no component-test stack in `ui/`; state model extracted pure and tested, component a thin map over it — flo's `PracticeSelector` as the shape reference).

## implementation, stage 3 — the selector

- `ui/src/selector.ts`: `selectorOptions(index, current)` — the pure model. Available projects by name; unavailable ones present and disabled with the diagnostic in the label (the spec's own rendering: `anpheq — roadmap directory not found`) and as the tooltip; an unknown current (URL names no configured project) rides as a disabled entry so the header reflects the URL truthfully. `projectPath(name)` landed beside `projectFromPath` in `project.ts` as its dual.
- `App.tsx`: `ProjectSelector` — native `<select>` in the project name's header slot (flo's pattern), `window.location.assign(projectPath(name))` on change: switching is a navigation, never a state mutation. The inversion: `ProjectBoard` fetches the project index beside the board; only the *index* failing keeps the pre-board fatal page (daemon tier — nothing to select); a board failure now renders in a `.board-error` body region under the live header, unknown and unavailable projects alike, so a broken default never strands the operator. `ItemModal` render gated on `board` presence (body-region error means no orderVersion to hand it).
- `style.css`: `.project-selector` as a quiet control (transparent until hover, carries the h1's weight, warn-colored disabled options), `.board-error` padding.
- Tests: `selector.test.ts` — available naming, unavailable flag+diagnostic, the trap case (unavailable default + healthy sibling: current present-disabled-flagged, sibling enabled, `projectPath` navigation), unknown-current phantom entry. 31 UI tests total; full Go suite + vet + `make build` green; live smoke (index + SPA route) good. `document.title` confirmed reading the configured name (follows `board.project`, which stage 2 made the configured name).

Terminus on the stage: clean, zero findings. Held for Michael's review before stage 4; approved.

## implementation, stage 4 — the daemon

- Dependency: `github.com/michaelquigley/dfw v0.1.0`, tray subpackage only (transitives: `fyne.io/systray`, dbus). `CGO_ENABLED=0 go build` confirmed clean — no webview headers anywhere near the binary.
- `cmd/ranger/daemon.go`: `ranger daemon`. `daemonBootstrap(path)` validates the config once — fail-fast naming the path and the fix — then assembles the shared `buildMux` over a file-backed source that `config.Load`s per request; deliberately no per-root load gate. `resolvePort(cfg, flag, changed)` encodes the precedence on cobra's `Changed` (an explicit `--port 4114` still wins). The `Listen` closure binds `127.0.0.1:{port}` and captures the bound listener's address into the board URL — the tray always points where the listener actually bound, immune to later `port:` edits. `tray.DaemonApp`: AppID `com.michaelquigley.ranger`, title `ranger`, the mark PNG, `SpawnWindow` deliberately unset (no webview, ever), one lowercase `open board` TrayItem plus dfw's built-in quit. Config path via `os.UserConfigDir` (= `~/.config/ranger/config.yaml` on linux).
- `cmd/ranger/browser.go`: `browserArgv(goos, url)` — `xdg-open` on linux, `rundll32.exe url.dll,FileProtocolHandler <url>` on windows (entry point in the argv), plain refusal naming the URL elsewhere; `openBrowser` starts the opener and reaps it (a long-lived daemon accumulates no zombies).
- Icon: `cmd/ranger/tray-icon.png`, 32px, rendered from `favicon.svg` via sharp/libvips (masks render correctly; oksvg couldn't), committed and `go:embed`ded. Render verified visually — the binoculars silhouette carries at tray size.
- Tests: bootstrap fail-fast (missing/invalid/empty config, each naming the path), the bootstrap-boundary mirror (broken root constructs, then reports unavailable over the assembled mux — pinning that root health never creeps into bootstrap), fresh-source visibility (a config edit lands on the very next request, no restart), port precedence including the explicit-default-value case, and per-platform browser argv (the windows branch never runs under a green linux build). Live check: `ranger daemon` with no config fail-fasts naming path + fix. The tray loop itself is the by-hand verification — no headless tray.

Green: full Go suite + vet, 31 UI tests, `make build`, `make headless`. Terminus on the stage: clean, zero findings. Held for Michael's review — the tray loop verification (icon render in a real tray, open board, quit) is his by-hand step — before stage 5.

Michael's review nit (cosmetic, left to my judgment): the bootstrap wrap appended "must name at least one project root" to *every* config error, pointing slightly wrong on a duplicate name or bad override. Fixed by splitting the hint on an `os.Stat`: a missing file gets "create … naming at least one project root", everything else gets "fix … and restart" (the raw load error already names its own repair; dd's `FileError` has no `Unwrap`, hence stat over `errors.Is`). Test now pins that an existing-but-broken config never gets the create hint. cmd tests + build green; live fail-fast re-verified.
